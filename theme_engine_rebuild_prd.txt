# AnimeSorter 테마 엔진 재구축 PRD

## 프로젝트 개요
**제목**: AnimeSorter 테마 엔진 재구축
**목적**: 하드 코딩된 스타일을 제거한 후, 중앙화된 테마 시스템을 구축하여 일관된 UI/UX 제공
**우선순위**: 높음 (UI 스타일링 시스템 복구)

## 비즈니스 목표
- 사용자 경험 향상: 일관된 시각적 디자인으로 사용자 만족도 증대
- 유지보수성 개선: 중앙화된 테마 관리로 개발 효율성 증대
- 접근성 강화: 고대비, 다크/라이트 테마 등 다양한 사용자 요구 충족
- 확장성 확보: 새로운 테마 추가 및 커스터마이징 용이성

## 현재 상황 분석
- 완료: 15개 GUI 컴포넌트에서 하드 코딩된 스타일 제거
- 누락: 테마 엔진 파일들 9개 (manager.py, types.py, token_loader.py 등)
- 위험: 현재 테마 시스템이 작동하지 않아 기본 스타일만 적용됨

## 기능 요구사항

### 핵심 기능
1. **테마 관리 (ThemeManager)**
   - 다크/라이트/자동 테마 전환
   - 시스템 테마 자동 감지 (Windows/macOS/Linux)
   - 실시간 테마 변경 및 적용

2. **토큰 시스템 (TokenLoader)**
   - 색상, 폰트, 간격 등의 디자인 토큰 관리
   - CSS 변수와 유사한 구조
   - 테마별 토큰 값 매핑

3. **템플릿 시스템 (TemplateLoader)**
   - QSS(Qt Style Sheets) 템플릿 로딩
   - 동적 테마 적용
   - 컴포넌트별 스타일 분리

4. **색상 유틸리티 (ColorUtils)**
   - 색상 변환 및 조정
   - 접근성 기준 충족 색상 생성
   - 투명도 및 블렌딩 지원

5. **아이콘 관리 (IconManager)**
   - 테마별 아이콘 자동 전환
   - SVG 아이콘 지원
   - 고해상도 디스플레이 대응

### 고급 기능
6. **접근성 옵션 (A11yOptions)**
   - 고대비 모드
   - 폰트 크기 조정
   - 색상 대비 강화

7. **검증 시스템 (Validation)**
   - 테마 파일 유효성 검사
   - 색상 대비 WCAG 기준 준수 확인
   - 오류 시 기본 테마로 폴백

8. **템플릿 컴파일러 (TemplateCompiler)**
   - 동적 QSS 생성
   - 조건부 스타일 적용
   - 성능 최적화

## 기술 요구사항

### 아키텍처
- MVVM 패턴: ViewModel과 View 분리
- 의존성 주입: 느슨한 결합으로 테스트 용이성
- 이벤트 기반: 테마 변경 시그널/슬롯 시스템
- 플러그인 구조: 확장 가능한 테마 시스템

### 기술 스택
- Python 3.10+: 타입 힌트 및 최신 기능 활용
- PyQt5: Qt 스타일시트 및 팔레트 시스템
- Pydantic: 설정 및 검증
- Logging: 구조화된 로깅 시스템

### 성능 요구사항
- 테마 전환: 100ms 이내 완료
- 메모리 사용량: 기존 대비 20% 이내 증가
- 시작 시간: 애플리케이션 시작 시 200ms 이내 테마 적용

## 파일 구조
```
src/gui/theme/
├── __init__.py                 # 테마 시스템 진입점
├── engine/                     # 핵심 엔진
│   ├── __init__.py
│   ├── manager.py             # 테마 관리자
│   ├── types.py               # 타입 정의
│   ├── token_loader.py        # 토큰 로더
│   ├── template_loader.py     # 템플릿 로더
│   ├── color_utils.py         # 색상 유틸리티
│   ├── icon_manager.py        # 아이콘 관리자
│   ├── template_compiler.py   # 템플릿 컴파일러
│   └── validation.py          # 검증 시스템
├── themes/                     # 테마 정의
│   ├── __init__.py
│   ├── dark.qss               # 다크 테마
│   ├── light.qss              # 라이트 테마
│   ├── high_contrast.qss      # 고대비 테마
│   └── tokens/                # 토큰 정의
│       ├── dark.json
│       ├── light.json
│       └── high_contrast.json
├── assets/                     # 테마별 자산
│   ├── icons/
│   │   ├── dark/
│   │   └── light/
│   └── images/
└── tests/                      # 테스트 코드
    ├── test_manager.py
    ├── test_token_loader.py
    └── test_integration.py
```

## 구현 단계

### Phase 1: 기본 구조 (1-2일)
1. 기본 클래스 구조 및 타입 정의
2. 테마 관리자 기본 기능
3. 단순한 다크/라이트 테마 적용

### Phase 2: 토큰 시스템 (2-3일)
1. 디자인 토큰 구조 설계
2. JSON 기반 토큰 로딩
3. 색상 및 폰트 토큰 적용

### Phase 3: 템플릿 시스템 (3-4일)
1. QSS 템플릿 로더 구현
2. 동적 스타일 적용
3. 컴포넌트별 스타일 분리

### Phase 4: 고급 기능 (2-3일)
1. 접근성 옵션 구현
2. 아이콘 관리자 구현
3. 검증 시스템 구현

### Phase 5: 통합 및 테스트 (2-3일)
1. 기존 GUI 컴포넌트와 통합
2. 성능 최적화
3. 종합 테스트 및 버그 수정

- 품질: 테스트 커버리지 90% 이상, MyPy 오류 0개
